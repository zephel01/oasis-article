# note / Qiita / Zenn を 1 つのリポジトリで書き分けるツールを作った

<!-- 上の H1 はタイトル用の仮置き。note の編集画面では H1 は書かず、
     タイトル欄にこの文字列を入れる。本文は下の導入から始める。 -->

---

最近、同じネタを note に書くか、Qiita に書くか、Zenn に書くかで、やたらと迷うようになった。

note 用に柔らかく書き直し、Qiita 用に検索に刺さる硬めの文にし、Zenn 用にコード中心で組み直す。1 回書いたものを別媒体にコピペすると、微妙に合わない。そのままだと「note だけど技術的すぎる」「Zenn だけど堅苦しい」といった座りの悪い記事になる。

かといって、ChatGPT や Claude に「これを Qiita 風にして」と頼むと、今度は別の AI 臭が乗ってくる。「いかがでしたか？」「本記事では〜ご紹介します」みたいなアレである。

そこで、3 媒体ぶんのワークスペースと、ローカル LLM で文体をリライトする CLI を 1 つのリポジトリにまとめた。名前は **oasis-article** にした。公開しているので、同じ悩みを持っている人がいたら試してみてほしい。

https://github.com/zephel01/oasis-article

## 何ができるのか

ざっくり 3 つ。

1. note / Qiita / Zenn 用のワークスペースを 1 つのリポジトリに同居させる
2. 下書きをローカル LLM (Ollama) で自分の文体にリライトする
3. リライト時に AI 臭のパターン 40 個超を検出して消す

ポイントは、全部ローカルで動くこと。外部の API に下書きを送りたくない派なので、そこは外せなかった。

## 構成

リポジトリはこうなっている。

```
oasis-article/
├── tools/    … 共通のローカル LLM ツール群
├── note/     … note.com 記事ワークスペース
├── qiita/    … Qiita CLI ワークスペース
└── zenn/     … Zenn CLI ワークスペース
```

`tools/` の中身は Python の CLI が何本か。メインは `article_rewriter.py` というリライトツールで、これに `--platform note` `--platform qiita` `--platform zenn` のどれかを渡すと、そのプラットフォームに合う形で書き直してくれる。

## なぜローカル LLM なのか

理由は 3 つある。

**1 つめは、下書きを外に出したくないから**。書き途中の記事って、まだ未整理で恥ずかしい。人に見られる前に API 経由でクラウドに送るのも、気分的に居心地が悪い。Ollama なら自分のマシンの中で完結する。

**2 つめは、自分の文体にチューニングできるから**。クラウド API は最新モデルを使える代わりに、「そのモデルの癖」が強く出る。手元のモデルなら、スタイルテンプレを当てて何度も回して、出力を自分好みに揃えやすい。

**3 つめは、お金がかからないから**。書きかけの記事を延々と回すのに、毎回 API トークンを消費するのはちょっと違う気がしていた。

## 使うモデル

32 GB までのメモリで動くのが現実ラインだと思っている。肌感としてこんな感じ。

- 8 〜 16 GB なら Qwen2.5 7B か Swallow 8B か Gemma 2 9B
- 16 〜 24 GB なら Qwen2.5 14B が第一候補
- 24 〜 32 GB なら Qwen2.5 32B (Q4) か Gemma 2 27B (Q5)

迷ったら 14B から試すのがいい。命令追従 (スタイルテンプレの遵守) が強めで、長文でも文体が崩れにくい。日本語の自然さで選ぶなら Gemma 2 27B のほうが頭ひとつ抜けている印象。

もちろんモデルは日進月歩なので、ここは半年後には総入れ替えになっているかもしれない。

## AI 臭 40+ パターンを消す

個人的に今回作っていて一番楽しかった部分がこれ。

AI が生成した日本語には、共通して現れる言い回しがある。たとえばこのあたり。

- 「〜と言えるでしょう」
- 「本記事では〜をご紹介します」
- 「いかがでしたか？」
- 「それでは〜見ていきましょう」
- 「〜することが可能です」
- 「〜を活用することで」
- 「〜は非常に重要です」
- 「お役に立てれば幸いです」

これらを 40 個以上、正規表現で検出して「ここ AI 臭いよ」と指摘する `--deai` モードを作った。LLM は呼ばず、パターンマッチだけで動くので一瞬で終わる。

```
python tools/article_rewriter.py draft.md --deai
```

ChatGPT / Claude / Gemini / Grok それぞれに特有の癖もテーブルで持っている。Claude だとやたらと「素晴らしい質問ですね」と言ってくるし、ChatGPT だと「以下にまとめました」が多い。ああいう、AI ごとの顔が透けて見える表現を狩るのが、個人的にちょっと爽快だった。

## 3 媒体に振り分ける

1 本の下書きから、3 媒体ぶんの最終原稿を作るのはこんな流れ。

```
python tools/article_rewriter.py draft.md --platform note  --output note/articles/draft
python tools/article_rewriter.py draft.md --platform qiita --output qiita/public/draft
python tools/article_rewriter.py draft.md --platform zenn  --output zenn/articles/draft
```

`--platform` を渡すと、ツールは次のことをする。

- プラットフォーム別のスタイルテンプレを自動で読み込む
- システムプロンプトに「Zenn なら `:::message` 使っていい」「note だと `:::message` は効かないから使うな」みたいな指示を差し込む
- 結果として、媒体の記法ルールを守った形で出力してくれる

これで「Zenn の `:::message` を note にそのまま貼って壊れる」みたいな事故が減った。

## やってみた感想

ツールができてから、書き方が少し変わった。以前は「この話はどこに書くか」を先に決めてから書き始めていたけど、今は先にざっと書いてから媒体を決めるようになった。媒体ごとの最適化は後から自動でかかるので、「書きたいものを書く → 後で振り分ける」という順番になる。

あと、自分で書いた文章を自分で `--deai` にかけると、わりと恥ずかしい。AI にリライトさせていないのに、自分の文章に「〜と言えるでしょう」が混ざっていることがある。いわゆる AI 臭は、AI だけのものじゃない。

## まとめ

- 3 媒体の執筆ワークスペースを 1 つのリポジトリにまとめた
- リライトはローカル LLM (Ollama) で完結、外部に送らない
- AI 臭 40+ パターンを検出して消すモードあり
- `--platform` で note / Qiita / Zenn ごとに最適化できる

同じように複数媒体で書いている人の役に立てば嬉しい。コードは GitHub で公開しているので、気軽に clone してみてほしい。
