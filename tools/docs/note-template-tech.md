<!-- ============================================================
  Note記事テンプレート集（技術系）

  ■ Note Markdown制約ルール（必ず守ること）

  【使える記法】
  - 見出し：## (h2) と ### (h3) のみ。# (h1) や #### (h4) は使えない
  - 太字：**テキスト**
  - 斜体：*テキスト*
  - 取り消し線：~~テキスト~~
  - 箇条書き：- または * で統一（混在NG）
  - 番号付きリスト：1. + 半角スペース
  - コードブロック：``` で囲む（言語指定なし推奨）
  - 引用：> テキスト
  - リンク：[テキスト](URL)
  - 水平線：---

  【使えない・崩れる記法】
  - テーブル（| --- | --- |）→ リスト形式で代替
  - # (h1)、#### (h4以下) → ## か ### か太字で代替
  - リスト内のコードブロック → リストの外に独立させる
  - ネストされたリスト → フラットにする
  - 画像のMarkdown記法 → Noteエディタから直接アップロード
  - 脚注、数式 → 非対応
  - コードブロックの言語指定（```bash）→ ``` のみ推奨
  - 全角スペースの混入 → 半角スペースに統一

  【書き方のコツ】
  - コードブロックの前後に必ず1行空行を入れる
  - リストの前後にも1行空行を入れる
  - 番号付き手順は「**Step 1：**」のように太字テキストで書く
  - 表の代わりに「**項目名** — 内容」のリスト形式を使う
  - URLは文中にベタ書きでもOK（Noteが自動リンクにする）

============================================================ -->
