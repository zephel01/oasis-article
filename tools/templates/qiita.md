# Qiita 向け 文体テンプレート

Qiita は「Google 検索から読まれる技術情報」の場。読者は「答えだけ早く欲しい」ケースが多いので、結論先出し・セクション小分け・エラーメッセージそのまま転記が効く。

## 文体の方針

- ですます体で統一。口語ほどカジュアルにしない（note より一段フォーマル寄り）
- 一文は 50〜70 文字を目安に。冗長な修飾は削る
- 一人称は省略。必要なら「筆者」ではなく「自分」
- 技術用語の初出ではカタカナと原語を併記（例: 「フック (hook)」）
- 箇条書き・表・コードブロックを積極的に使って読みやすく

## Qiita 記法の使い方

- **注意ボックス**: `> **Note**: ...` / `> **Warning**: ...`
- **折りたたみ**: `<details><summary>…</summary> 本文 </details>`
- **コードブロックは言語指定 + ファイル名**: ```` ```python:main.py ```` のように
- **差分**: ```` ```diff ```` で `+` / `-` 行を示す
- **目次**: 必要なら `[[toc]]`
- 数式は KaTeX (`$E=mc^2$`, `$$...$$`)

## 構成テンプレ（技術解説）

1. TL;DR（3〜5行で結論）
2. 背景 / なぜ書くか
3. 前提条件
4. 実装
5. 動作確認
6. ハマりどころ（details 折りたたみ）
7. まとめ
8. 参考リンク

## 除去する AI 臭

- 「〜と言えるでしょう」→ 断定
- 「本記事では〜解説します」→ 削除
- 「〜が求められます」→「〜が必要」
- 「〜を実現しました」→「〜した」「〜できた」
- 「いかがでしたか？」→ 削除
- 「〜について解説します」→ 削除
- 「〜することが可能です」→「〜できる」

## SEO を意識する

- 見出しに検索されそうなキーワードを自然に含める
- エラーメッセージは原文のまま転記（検索ヒット対策）
- コードは実行可能な形で示す（擬似コードは避ける）
