# Zenn 向け 文体テンプレート

Zenn は「コード中心 + Markdown を表現力豊かに使える」プラットフォーム。Qiita より若干カジュアルに書いてもよく、図解や :::message などの装飾で情報の階層を明確にできる。

## 文体の方針

- ですます体で統一。適度な口語を許容（Qiita よりやわらかく）
- 一文は 60 文字前後。コードや図のあとは短文で説明
- 結論先出し + 「なぜそれが大事か」のコンテキストを添える
- 技術用語の初出は英語の原語を示す（例: 「イベントループ (Event Loop)」）

## Zenn 記法の使い方

- **メッセージボックス**: `:::message` / `:::message alert` / `:::`
- **アコーディオン**: `:::details タイトル` / `:::`
- **コードブロックは言語指定 + ファイル名**: ```` ```ts:src/index.ts ````
- **差分コードブロック**: ```` ```diff ts ```` で `+` / `-` を示す
- **リンクカード**: URL を1行で置くだけ（`https://zenn.dev/...`）
- **埋め込み**: `@[tweet](URL)` `@[youtube](ID)` `@[card](URL)` `@[gist](URL)`
- **脚注**: `本文[^1]` / `[^1]: 注釈`
- **目次**: `@[toc]`
- 数式は KaTeX (`$...$`, `$$...$$`)

## 構成テンプレ（技術解説）

1. リード文（1〜3段落、自分ごと化）
2. :::message で「この記事で学べること」を箇条書き
3. 背景 / なぜこれを書くのか
4. 前提条件
5. 結論（忙しい人向け）
6. 問題
7. 解決方法（案A / 案B）
8. 動作確認
9. ハマりどころ（:::details）
10. まとめ
11. 参考（リンクカード）

## 除去する AI 臭

- 「〜と言えるでしょう」→ 断定
- 「本記事では〜解説します」→ 削除
- 「〜が求められます」→「〜が必要」
- 「〜を実現しました」→「〜できた」
- 「いかがでしたか？」→ 削除
- 「〜について解説します」→ 削除
- 「〜することが可能です」→「〜できる」
- 「〜を活用することで」→「〜を使えば」

## その他

- 画像は `/images/<slug>/foo.png` のように slug 配下に置く
- 画像下のキャプションは `*キャプション*` でイタリックに
- コードは copy-paste で動く状態を目指す（Zenn の読者は試す人が多い）
