# note 向け 文体テンプレート

note.com は「個人の文体で読ませるプラットフォーム」。論文や技術マニュアルっぽい硬さより、書いている人の顔が見える文章のほうがウケやすい。

## 文体の方針

- 口語調のですます体をベースに、適度にため口を混ぜる
- 「〜してみた」「〜ハマった」「〜できた」「〜だった」を積極的に使う
- 一人称は省略するか「自分」。「筆者」「私たち」は使わない
- 一文は60文字を超えないように。長くなったら分割
- 結論を先に、詳細を後に
- 段落の冒頭で「何の話か」が分かるようにする

## note 特有のポイント

- **見出しは ## と ### の2階層まで**（note の装飾は最小限）
- **Markdown の特殊記法は使わない**（:::message や <details> は note では効かない）
- コードブロックは使えるが、言語指定は任意
- 画像はキャプションを添える前提で書く（「↑〇〇の様子」など）
- 有料記事の区切りは書き手が後で入れる。構成時は意識しない

## 除去する AI 臭

（ChatGPT / Claude / Gemini / Grok 共通）

- 「〜と言えるでしょう」→ 断定か感想
- 「本記事では〜ご紹介します」→ 削除して本題からいきなり
- 「はじめに」の教科書的な導入 → 自分の体験談から始める
- 「〜が求められます」→「〜が必要」
- 「〜を実現しました」→「〜できた」
- 「いかがでしたか？」→ 全削除
- 「それでは〜見ていきましょう」→ 全削除
- 「〜について解説します」→ 削除
- 「〜することが可能です」→「〜できる」
- 「〜を活用することで」→「〜を使えば」
- 「〜は非常に重要です」→「〜が大事」

## 締め方

- 教科書的な「まとめ」ではなく、感想・次にやりたいこと・読者への問いかけで閉じる
- 「参考になれば幸いです」「お役に立てれば幸いです」は禁止
- いきなり書き終えてもよい
